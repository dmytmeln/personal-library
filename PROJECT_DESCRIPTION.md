# Детальний опис бекенду: Personal Library (library-app)

## 1. Загальна архітектура
Бекенд-частина проєкту **Personal Library** побудована на базі фреймворку **Spring Boot 3.4.4** та використовує **Java 17**. Архітектура застосунку слідує принципам модульного моноліту з чітким розділенням відповідальності за доменами.

### Основні архітектурні патерни:
- **Controller-Service-Repository**: Класичний підхід для розділення логіки обробки запитів, бізнес-логіки та доступу до даних.
- **DTO (Data Transfer Objects)**: Використовуються для передачі даних між шарами та через API, що дозволяє приховати внутрішню структуру сутностей бази даних.
- **MapStruct**: Автоматизоване мапування між сутностями (Entities) та DTO для забезпечення високої продуктивності та чистоти коду.
- **Global Exception Handling**: Централізована обробка помилок за допомогою `@RestControllerAdvice`, що забезпечує узгоджені відповіді API у разі виникнення виключень.

## 2. Технологічний стек
- **Java 17**: Використання сучасних конструкцій мови (records, sealed classes, var).
- **Spring Data JPA**: Потужний інструмент для роботи з базою даних через Hibernate.
- **PostgreSQL**: Основна реляційна база даних.
- **Liquibase**: Система управління міграціями бази даних, що забезпечує версійність схеми БД.
- **Jakarta Bean Validation**: Валідація вхідних даних на рівні DTO.
- **Lombok**: Мінімізація шаблонного коду (генерація геттерів, сеттерів, конструкторів).
- **Maven**: Система збірки та управління залежностями.

## 3. Безпека (Security)
Система безпеки реалізована за допомогою **Spring Security** з використанням безсесійного (stateless) підходу на базі **JWT (JSON Web Tokens)**.

- **JWT у Cookies**: Токени доступу та оновлення зберігаються у HTTP-only cookies для захисту від XSS-атак.
- **Role-Based Access Control (RBAC)**: Розмежування прав доступу для ролей `USER` (звичайний користувач) та `ADMIN` (адміністратор контенту).
- **CORS**: Налаштована політика обміну ресурсами між різними доменами для безпечної взаємодії з Angular-фронтендом.
- **BCrypt**: Надійне хешування паролів користувачів перед збереженням у БД.

## 4. Доменні модулі та функціональність
Бекенд розділений на наступні ключові модулі:

### Управління книгами та авторами (`book`, `author`, `category`)
- Глобальний каталог книг, доступний для всіх користувачів.
- Можливість перегляду детальної інформації про авторів та жанри (категорії).
- Адмін-панель для додавання та редагування контенту.

### Особиста бібліотека користувача (`library_book`, `library_author`, `library_category`)
- Можливість додавати книги з глобального каталогу до власної бібліотеки.
- Трекінг статусу читання (прочитано, читаю, планую).
- Система оцінювання книг та написання власних відгуків.

### Колекції та нотатки (`collection`, `note`)
- Створення користувацьких колекцій для групування книг.
- Додавання персональних текстових нотаток до кожної книги в бібліотеці.

### Цілі читання та статистика (`reading_goal`, `statistics`)
- Встановлення річних або місячних цілей за кількістю книг.
- Розрахунок прогресу та візуалізація статистики читання користувача.

## 5. Просунуті можливості: Recommendation Engine
Однією з ключових особливостей є система персоналізованих рекомендацій (`recommendations`), яка використовує елементи машинного навчання:

- **Векторний пошук**: Використання **Cosine Similarity** у PostgreSQL для пошуку схожих книг.
- **TF-IDF Векторизація**: Книги перетворюються на вектори на основі їхніх описів за допомогою бібліотеки `smile-nlp`.
- **Профілювання користувача**: Система будує вектор інтересів користувача на основі книг у його бібліотеці.
- **Hibernate Vector**: Інтеграція для ефективної роботи з векторними даними в JPA.

## 6. Моніторинг та DevOps
- **Spring Boot Actuator**: Ендпоінти для перевірки стану здоров'я (`health`) та метрик.
- **Micrometer + Prometheus**: Збір та експорт метрик продуктивності.
- **Loki + Logback**: Система централізованого логування.
- **Docker Compose**: Конфігурація для швидкого розгортання всього оточення (БД, Prometheus, Grafana).

## 7. Локалізація (i18n)
- Підтримка багатомовності (українська та англійська) на рівні серверних повідомлень про помилки та фільтрації контенту залежно від локалі користувача.
